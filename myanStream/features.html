完整的直播電商功能模組 - 參考 Jambo 等業界標準

🎯 功能總覽

💡 myanStream 核心價值

myanStream 是一套完整的直播電商 SaaS 解決方案，整合多平台直播推流技術與完整電商功能，從直播場次管理、即時收單、訂單處理到金流物流，提供全方位的直播帶貨體驗。

📊 功能模組架構

A. 串流/直播

多平台直播推流（YouTube/Facebook/Instagram），SRS + FFmpeg 技術實現一推多發

B. 收單整合

12 種收單渠道（方案分級選配），支援直播聊天室、LINE 群組、FB 社團等多元收單方式

C. 商品管理

SPU/SKU 規格管理、庫存管理、價格策略、候補單機制

D. 訂單系統 OMS

收單池管理、訂單狀態流轉、批量操作、訂單追蹤

E. 金流

藍新、綠界、Stripe 等金流整合，支援信用卡、ATM、超商代碼

F. 物流出貨

黑貓、新竹、超商取貨等物流整合，批量產生托運單

G. 訊息/客服

多平台訊息匯流、罐頭訊息、客服自動化助理

H. 會員 CRM

會員管理、標籤分群、購買記錄、會員等級

I. 行銷活動

優惠券、推播通知、EDM、開播提醒

🔗 模組之間的關聯性

  • 多元收單渠道：提供 12 種收單方式（方案分級選配，如 12 選 5），涵蓋直播聊天室、LINE 群組、FB 社團、Messenger、商城等。直播時和非直播時都可持續收單。

📡 A. 串流/直播

💡 核心功能

提供多平台直播推流功能，主播只需推一路流到系統，即可自動轉推到 YouTube、Facebook、Instagram 等多個平台，節省頻寬與操作成本。

1️⃣ 多平台推流

YouTube Live

自動呼叫 YouTube API 建立直播間，取得 RTMPS 推流位址

Facebook Live

整合 Facebook Graph API，支援粉絲專頁與個人帳號直播

Instagram Live

透過 Instagram API 推送直播流（需平台授權）

TikTok Live（待規劃）

TikTok 直播整合功能，依平台 API 開放程度規劃

2️⃣ 技術架構

🏗️ SRS + FFmpeg 架構

  • SRS 媒體伺服器：接收主播的 RTMP 推流，觸發 Webhook 通知
  • FFmpeg 轉推引擎：從 SRS 拉取流並轉推到各平台 RTMPS
  • 按需啟動：每場直播獨立啟動 FFmpeg 實例，直播結束後自動停止
  • 容錯機制：單一平台失敗不影響其他平台

3️⃣ 直播場次管理

功能 說明
建立直播 設定直播標題、描述、封面圖，選擇推流平台
推流設定 自動生成 RTMP URL 與 Stream Key，支援 OBS 或 App 內建推流
直播狀態 READY / STARTING / PUBLISHING / ENDED / FAILED
直播回放 自動保存直播錄影（選配功能）

⚠️ 重要技術說明

第三方平台的聊天室 API 主要都是使用輪詢（Polling）方式，而非 Webhooks。需要定時呼叫平台 API 拉取新留言（例如每 2-5 秒一次）。

2️⃣ API 拉取機制

方式 說明 適用平台

3️⃣ 收單流程

📝 從留言到訂單的完整流程

  1. 觀眾留言：「+1 紅色 M」、「我要藍色 L x2」
  2. 系統解析：自動識別商品編號、規格（顏色/尺寸）、數量
  3. 建立暫存單：加入收單池，等待主播確認
  4. 主播確認：檢查庫存、買家資訊，確認或修改
  5. 生成訂單：轉為正式訂單，發送通知給買家

💡 方案分級選配（X 選 Y 模式）

不同方案可選用不同數量的收單渠道：

  • 入門版：6 選 1（選擇 1 種收單方式）
  • 標準版：8 選 3（選擇 3 種收單方式）
  • 專業版：12 選 5（選擇 5 種收單方式）
  • 企業版：12 種全開（所有收單方式）

📦 C. 商品管理

4️⃣ 候補單（缺貨登記）

💡 當商品缺貨時的解決方案

熱門商品快速售罄時，買家可以登記候補，等商品到貨後系統自動通知，避免流失客戶，並能預估補貨需求。

  • 缺貨登記：買家可登記候補，填寫數量與規格
  • 自動通知：補貨後自動發送通知給候補名單
  • 優先購買：候補客戶享有優先購買權（限時）
  • 需求預估：根據候補數量決定補貨數量

1️⃣ 收單池管理
